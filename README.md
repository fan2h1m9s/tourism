##智能旅游Agent系统项目背景（供AI coding使用）
随着旅游消费升级，游客对旅游服务的个性化、实时性、精准性需求日益提升，传统旅游规划工具存在场景适配不足、信息滞后、交互生硬等问题，无法满足用户单对话多行程规划、实时景点查询、动态行程调整等复杂需求。为解决上述痛点，依托LangChain4j、Spring Boot、MCP、Redis、MySQL等核心技术栈，启动智能旅游Agent系统开发，旨在构建一个基于大模型的智能交互系统，为游客提供全流程个性化旅游服务。
当前旅游类智能服务存在三大核心技术瓶颈：一是多行程复杂需求处理能力不足，单对话中包含多个行程规划的场景下，模型易出现上下文混乱、幻觉率偏高的问题，影响服务准确性；二是会话上下文管理效率低，传统上下文存储方式无法兼顾读写速度与内存占用，导致交互延迟过高，影响用户体验；三是实时数据接入与异常处理不完善，行程规划依赖静态数据易出现与实际景点情况脱节的问题，同时非法输入、不合规输出等情况难以有效管控，降低系统可靠性。
本项目核心目标是通过技术优化与架构设计，解决上述瓶颈，具体依托LangChain4j实现大模型交互与任务拆分，借助Spring Boot搭建稳定高效的后端架构，集成高德MCP服务获取实时景点（POI）数据，利用Redis实现高性能会话上下文存储，通过MySQL存储系统核心业务数据。通过引入轻量级任务拆分Agent、自定义ChatMemoryStore、搭建Guardrails护栏等技术方案，实现个性化旅游路线规划、动态POI检索、实时交互查询等核心功能，提升系统的准确性、高效性与合规性，为游客提供更智能、便捷的旅游服务支持，同时为后续功能迭代与技术优化奠定坚实基础，助力AI技术在旅游场景的落地应用。

当前放在简历的内容如下：
【技术栈】
LangChain4j、Spring Boot、MCP、Redis、MySQL
【项目介绍】
本项目是一个面向游客的智能旅游 Agent 系统，旨在通过大模型提供个性化的旅游路线规划、动态景点（POI）检索与实时交互查询服务。
【功能描述】
(1) 在主模型前引入轻量级任务拆分Agent，将单对话多行程的非常规场景需求自动拆解为独立子任务，临时AI实例处理不同子任务上下文，上下文Token压缩率达42%，多行程场景模型幻觉率降低15%。
(2) 基于Redis自定义ChatMemoryStore，结合滑动窗口算法控制上下文长度，会话读写P99延迟小于100ms。
(3) 集成高德MCP服务，通过Tool Use接入实时数据，将行程生成幻觉率从22%降至4.5%，让规划结果更符合真实场景。
(4) 搭建Guardrails护栏+异常兜底重试机制，处理非法输入输出，非法输入拦截率达99%，输出格式合规率从81%提升至98.3%。
